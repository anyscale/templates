You are given the name of ONE Anyscale workspace with a pinned GPU worker group (a GPU
instance type with min_nodes > 0). The name is in $TEST_WORKSPACE, cloud aws-public-us-west-2.

Goal: make that GPU worker group scale to zero — set min_nodes = 0, keep max_nodes as-is,
change NOTHING else. On Anyscale a RUNNING workspace cannot have its compute config changed,
so this is a real terminate -> update -> start (~3-5 min restart). Be terse; log every
command and its output.

ABSOLUTE RULES:
  - Operate ONLY on the workspace whose name == $TEST_WORKSPACE. Never touch another.
  - Change ONLY min_nodes (-> 0) on GPU worker group(s). Keep everything else identical:
    head node, instance types, max_nodes, market type, group names, image, env, flags.
  - You MUST run validate_config_diff.py and see "PASS" BEFORE you terminate/update anything.
    If it prints any BAD line or FAIL, STOP and report — change nothing.
  - Never delete a workspace. (terminate here = stop the cluster; the workspace itself
    persists and /mnt storage survives.)

STEPS:

1. RESOLVE — find the workspace, read its live compute config AND image:
     anyscale workspace_v2 list --no-interactive --json --cloud aws-public-us-west-2
       -> the single entry whose name == $TEST_WORKSPACE; note its id. Zero or >1 -> STOP.
     anyscale workspace_v2 get --id <id> --json --verbose
       -> config.compute_config (inline: head_node, worker_nodes[]) and config.image_uri.
          RECORD image_uri — you must pass it back on update (step 4) or update fails with
          "both the build ID and compute template ID must be provided".

2. BUILD — write autozero.json = the live compute_config copied verbatim, with min_nodes set
   to 0 on the GPU worker group(s) only (GPU families: g*, p*, inf*, trn*). JSON is valid
   YAML, so a .json file works with -f.

3. VALIDATE (required gate — do not skip):
     python3 validate_config_diff.py <id> autozero.json
   Proceed ONLY on "PASS" (exit 0). Any BAD line / FAIL -> STOP, report, change nothing.

4. APPLY — terminate, update, start. Each must succeed before the next:
     anyscale workspace_v2 terminate --id <id>
     anyscale workspace_v2 wait --id <id> --state TERMINATED --timeout-s 480
     anyscale compute-config create -n <name>-autozero -f autozero.json
     anyscale workspace_v2 update <id> --compute-config <name>-autozero --image-uri <image_uri>
     anyscale workspace_v2 start --id <id>
     anyscale workspace_v2 wait --id <id> --state RUNNING --timeout-s 600

5. VERIFY:
     anyscale compute-config get <name>-autozero
   Confirm the GPU group shows min_nodes: 0 with max_nodes unchanged and nothing else changed.
   Print a short before/after (group name, min N->0, max unchanged) and the final state.
